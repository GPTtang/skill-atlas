# Internal Comms

帮助撰写各类企业内部沟通文档，使用清晰、专业的格式传达进展、计划和问题。

## 支持的文档类型

- **3P Updates**：Progress（进展）/ Plans（计划）/ Problems（问题）格式的团队周报
- **Company Newsletter**：公司全员简报
- **FAQ Responses**：常见问题解答
- **Status Reports**：项目状态报告
- **Leadership Updates**：面向管理层的更新
- **Incident Reports**：事故/障碍报告
- **Project Updates**：项目进度更新

## 流程

1. 识别用户需要的文档类型
2. 询问必要信息：时间范围、受众、核心内容要点
3. 选择合适的格式和语气（正式 vs. 轻松）
4. 起草文档，确保结构清晰、重点突出
5. 提供修改建议和替代表达

## 各文档格式

### 3P Update（进展/计划/问题）

```markdown
# [团队/项目名称] 周报 - [日期]

## ✅ Progress（本周进展）
- [完成的事项，量化结果]
- [里程碑达成]

## 📋 Plans（下周计划）
- [具体计划，附预期完成时间]
- [优先级排序]

## ⚠️ Problems（问题/风险）
- [当前阻塞点]
- [需要支持的事项]
```

### Status Report（状态报告）

```markdown
# [项目名] 状态更新 - [日期]

**状态**：🟢 正常 / 🟡 有风险 / 🔴 阻塞

**概要**：[1-2 句话说明整体进展]

## 本期完成
- [具体成果]

## 指标
| 指标 | 目标 | 实际 |
|------|------|------|
| [指标1] | [目标值] | [实际值] |

## 下一步
- [Action items，含负责人和截止日期]

## 风险/阻塞
- [风险描述，附缓解方案]
```

### Company Newsletter（全员简报）

```markdown
# [公司名] 简报 - [月份/时间]

**本期亮点**

[用轻松的语气分享重要新闻、成就和文化活动]

## 公司动态
- [重要公告]

## 团队成就
- [值得庆祝的里程碑]

## 即将发生
- [重要日期和活动]

---
感谢大家的努力！
```

## 示例

### 示例 1：3P Update

**输入**：帮我写一个工程团队的 3P 周报，本周完成了用户认证模块，下周要做支付集成，当前有一个第三方 API 文档不清晰的问题。

**输出**：

```markdown
# 工程团队周报 - 2026-03-14

## ✅ Progress
- 用户认证模块开发完成，通过全部 47 个测试用例，已部署到 staging 环境
- 代码覆盖率达到 92%，符合质量门禁要求

## 📋 Plans
- 本周启动支付集成模块开发（Stripe + 微信支付）
- 周五完成支付流程的技术方案评审

## ⚠️ Problems
- 第三方支付 API 文档存在歧义，已提交 support ticket（#1234），等待回复
- **需要支持**：如有支付集成经验的工程师，欢迎分享最佳实践
```

### 示例 2：Incident Report

**输入**：写一份事故报告，昨天下午 3 点到 4 点服务中断 1 小时，原因是数据库连接池耗尽。

**输出**：

```markdown
# 事故报告：服务中断 - 2026-03-13

**严重级别**：P1
**影响时长**：60 分钟（15:00 - 16:00）
**影响范围**：所有用户无法登录

## 根本原因
数据库连接池配置上限（50 个连接）在流量高峰期耗尽，导致新请求队列堆积，服务不可用。

## 处置过程
- 15:00 监控报警触发
- 15:12 工程师开始排查，定位到连接池问题
- 15:35 临时扩容连接池上限至 200
- 16:00 服务完全恢复

## 后续改进
- [ ] 将连接池上限调整为动态扩缩（本周完成）
- [ ] 增加连接池使用率监控告警（下周完成）
- [ ] 进行容量规划评审（本月完成）
```

## 写作原则

- **简洁优先**：管理层时间有限，用最少的字传达最关键的信息
- **量化结果**：用数字代替形容词（"完成了 3 个功能" 而非 "完成了很多功能"）
- **明确行动**：每个 Problem 和风险都要有对应的 next step 或求助对象
- **匹配受众**：面向全员的语气轻松，面向管理层的语气正式
